# WebUI Codex Shell 重构设计

日期：2026-07-12

## 目标

本阶段将 SpecGate WebUI 从当前的“三栏后台面板”调整为接近 Codex 桌面端的工作台界面。核心目标不是增加真实 LLM 能力，而是让已有的项目创建、对话运行、产物下载、运行审计、审批、设置等能力进入一个更自然的产品 Shell。

最终用户进入登录后的主界面时，应看到：

- 顶部菜单栏：`文件 / 编辑 / 帮助`，不包含 `视图`。
- 左侧项目栏：只保留 `新项目`、`搜索`、项目列表、底部 `设置`。
- 右侧主工作区：上方项目标题栏，中间对话流，下方输入栏。
- 运行详情入口：不再常驻右侧三栏面板，改为项目标题旁的 `...` 菜单进入。
- 键盘快捷键：支持用户指定的 Codex-like 快捷操作。

## 非目标

本阶段不做以下内容：

- 不接入真实 LLM。
- 不让前端硬编码根据用户主题生成真实页面。
- 不实现完整设置页面，只提供设置入口和占位视图。
- 不实现服务端关闭浏览器窗口；`退出` 只尝试关闭当前浏览器页签或窗口。
- 不做 ZIP 上传项目。项目导入先采用浏览器读取 `spec.md`、`checklist.md`、可选 `index.html` 的轻量方案。
- 不改动核心 Runner、安全策略、Mock LLM、审批、RAG、压缩、多代理隔离等后端能力。

## 当前问题

当前 WebUI 已经能完成登录、注册、项目创建、运行、审计查看、审批和下载产物，但界面呈现仍偏“管理后台”：

- 右侧详情面板长期占用空间，让对话区域不够集中。
- 项目创建仍主要是文本框输入，和“导入自己的项目文件夹/文件”的产品叙事不一致。
- 顶部没有稳定菜单栏，文件、设置、退出等操作分散在侧栏按钮中。
- 项目搜索、历史导航、侧栏折叠等工作台体验还没有形成。

本设计通过主 Shell 重构解决这些交互问题，同时复用现有后端 API 和多数前端渲染逻辑。

## 总体结构

登录后页面采用两层布局：

1. 顶部 `app-menu-bar`
   - 左侧是侧栏切换按钮、后退、前进。
   - 中间是菜单：`文件 / 编辑 / 帮助`。
   - 不展示项目状态或复杂指标。

2. 下方 `app-shell`
   - 左侧 `project-sidebar`。
   - 右侧 `workspace-main`。

`workspace-main` 再分三段：

- `workspace-titlebar`：左侧显示当前项目名称；项目名旁边有 `...` 菜单。
- `message-list`：居中显示用户消息、SpecGate 回复、运行摘要。
- `composer`：居中输入栏，用户输入任务并发送。

当没有选中项目时，右侧显示一个干净的空状态，提示用户新建或搜索项目，不展示营销式说明。

## 顶部菜单

顶部菜单包含 `文件 / 编辑 / 帮助`。

### 文件菜单

`文件` 下包含：

- `新窗口`，快捷键 `Ctrl+Shift+N`：调用 `window.open(window.location.href, "_blank")`。
- `新项目`，快捷键 `Ctrl+N`：打开新项目导入对话框。
- `关闭`，快捷键 `Ctrl+W`：关闭当前选中项目，回到无项目状态，不删除项目。
- 分隔线。
- `设置`，快捷键 `Ctrl+,`：进入设置占位视图。
- 分隔线。
- `登出`：调用现有登出 API，返回登录界面，并清空登录表单。
- `退出`，快捷键 `Ctrl+Q`：调用 `window.close()`；如果浏览器阻止关闭，则显示一条轻量提示。

### 编辑菜单

本阶段 `编辑` 菜单保留基础入口，不承载复杂编辑器能力：

- `搜索`，快捷键 `Ctrl+G`：打开项目搜索弹窗。

后续如需要，可在这里加入复制、粘贴、撤销等浏览器原生提示，但本阶段不做。

### 帮助菜单

`帮助` 用于展示项目说明入口，保持轻量：

- `关于 SpecGate`：打开一个简短弹窗，说明当前是 Mock-first harness 工作台。

## 键盘快捷键

本阶段支持以下快捷键：

- `Ctrl+B`：折叠或展开左侧栏。
- `Ctrl+N`：新项目。
- `Ctrl+G`：搜索项目。
- `Ctrl+Q`：退出当前页面。
- `Ctrl+,`：进入设置占位视图。
- `Ctrl+Shift+N`：新窗口。
- `Ctrl+W`：关闭当前项目。
- `Ctrl+[`：后退到上一个视图状态。
- `Ctrl+]`：前进到下一个视图状态。

快捷键处理原则：

- 当焦点在输入框或 textarea 内时，`Ctrl+Enter` 仍可作为后续扩展；本阶段不新增。
- `Ctrl+N`、`Ctrl+G`、`Ctrl+,`、`Ctrl+B` 即使焦点在输入区域，也应触发 Shell 操作，因为它们是全局导航命令。
- 触发浏览器默认行为风险较高的组合键需要 `preventDefault()`，例如 `Ctrl+N`、`Ctrl+W`。

## 视图历史

前进后退不是浏览器路由，而是 WebUI 内部视图历史。

定义视图状态：

- `conversation`：当前项目对话界面。
- `settings`：设置占位视图。
- `detail-status`：运行状态详情。
- `detail-preview`：产物预览/下载详情。
- `detail-report`：报告详情。
- `detail-audit`：审计详情。
- `detail-approvals`：审批详情。

前端维护：

- `state.view`：当前视图。
- `state.viewBackStack`：后退栈。
- `state.viewForwardStack`：前进栈。

调用 `pushView(nextView)` 时，将当前视图推入后退栈，并清空前进栈。点击后退时，从后退栈弹出目标视图，将当前视图推入前进栈。点击前进反向处理。

选择项目、关闭项目、登出时应重置视图历史，避免在不同用户或项目之间跳转到旧状态。

## 左侧栏

左侧栏只包含四类元素：

- 顶部按钮：`新项目`。
- 顶部按钮：`搜索`。
- 项目列表。
- 底部按钮：`设置`。

不保留当前侧栏中的用户账户文字、刷新按钮、右侧更新按钮、圆形蓝色“设”图标。

项目列表以紧凑卡片形式展示：

- 项目名称。
- 最近运行状态或无运行状态。
- 当前选中项目高亮。

项目数量较多时，项目列表区域滚动，顶部按钮和底部设置保持固定。

## 侧栏折叠与边缘唤起

`Ctrl+B` 或左上角侧栏按钮可折叠侧栏。

折叠后：

- 主对话区域占据屏幕主体，消息和输入栏保持居中。
- 页面最左边保留一个窄的 hover hot zone。
- 鼠标移动到左边界时，侧栏以浮层形式临时出现。
- 鼠标离开侧栏后，若用户没有显式展开，侧栏自动隐藏。

实现上使用 CSS class 控制：

- `body.sidebar-collapsed`
- `body.sidebar-peeking`

避免通过频繁 inline style 修改布局。

## 新项目导入

新项目弹窗从当前“手动输入文本框”调整为文件导入优先：

- 项目名称输入框。
- `spec.md` 文件选择，必填。
- `checklist.md` 文件选择，必填。
- `index.html` 文件选择，可选。

浏览器用 `File.text()` 读取文件内容，然后调用现有创建项目 API：

- `name`
- `spec_text`
- `checklist_text`
- `index_html`

项目名称为空时，可从 `spec.md` 文件名或默认 `新项目` 派生，但 UI 仍鼓励用户填写名称。文件类型不做过度限制，只校验必填文件存在和读取成功。

为了降低改动风险，本阶段不新增后端上传接口。

## 搜索项目

点击侧栏 `搜索` 或按 `Ctrl+G` 打开居中小弹窗：

- 背景轻微变暗。
- 弹窗顶部是搜索输入框。
- 下方显示匹配项目列表。
- 支持按项目名称和最近运行状态过滤。
- 点击结果后选择项目、关闭弹窗、进入对话视图。
- `Esc` 关闭弹窗。

搜索只作用于当前已加载项目列表，不新增服务端搜索接口。

## 主对话区

主对话区保持三段：

- 顶部标题栏：只显示项目名称和项目 `...` 菜单。
- 中间对话流：用户消息、SpecGate 回复、运行摘要在固定最大宽度内居中展示，两侧留白。
- 底部输入栏：固定最大宽度居中，按钮靠右，避免横向拥挤。

对话流继续复用现有 messages API。运行完成后仍展示摘要和下载入口，但更详细的运行证据从 `...` 菜单进入。

## 项目 `...` 菜单

项目标题旁的 `...` 菜单承载当前右侧详情面板的能力：

- `状态`
- `预览`
- `报告`
- `审计`
- `审批`

选择后进入对应 `detail-*` 视图。详情内容可复用现有函数：

- `renderStatusDetail`
- `renderPreviewDetail`
- `renderReportDetail`
- `renderAuditDetail`
- `renderApprovalsDetail`

区别是这些内容不再常驻右侧栏，而是在主工作区中作为当前视图展示。这样既保留研究证据，又不挤占日常对话空间。

## 设置入口

本阶段设置页面只做入口和占位视图：

- 顶部文件菜单 `设置`。
- 左侧栏底部 `设置`。
- 快捷键 `Ctrl+,`。

进入后主工作区显示设置占位视图，不在本阶段重新设计或完善设置页面。现有设置 API 和相关前端函数可以保留，但不作为本阶段验收重点。后续如果要继续做设置页，再单独设计 API Key、治理策略、上下文策略等配置体验。

## 登出与表单清理

点击 `登出` 后：

- 调用现有 `/api/auth/logout`。
- 清空用户、项目、消息、当前运行、审批、设置和视图历史状态。
- 返回登录界面。
- 清空登录表单中的用户名和密码。
- 清空登录错误消息。

这个行为修复之前“返回登录界面但表单可能残留”的产品问题。

## 视觉约束

界面风格参照 Codex，但不复制具体品牌资产：

- 色彩使用中性灰、白、黑和少量边框阴影。
- 主要区域保持安静、工具化，不做营销落地页。
- 卡片圆角不超过 8px。
- 主对话宽度稳定，避免文本横跨全屏。
- 菜单、弹窗、按钮状态需要有 hover/focus 样式。
- 移动端或窄屏下，侧栏默认可覆盖主区域，不让内容重叠。

## 实现边界

主要修改文件：

- `src/specgate/web_static/index.html`
- `src/specgate/web_static/styles.css`
- `src/specgate/web_static/app.js`
- `tests/test_web_static.py`

原则：

- 不改后端 API。
- 不改数据库 schema。
- 不改 Runner 行为。
- 优先重组静态 DOM 和前端状态。
- 保留现有运行、审计、审批、设置渲染函数，必要时调整挂载位置。
- 更新静态资源版本 query，避免浏览器缓存旧界面。

## 测试策略

静态测试需要覆盖：

- 顶部菜单存在 `文件 / 编辑 / 帮助`，且没有 `视图`。
- 文件菜单包含用户指定命令和快捷键文本。
- 存在侧栏折叠按钮、搜索按钮、设置入口。
- 新项目弹窗包含 `spec.md`、`checklist.md`、可选 `index.html` 文件输入。
- 搜索弹窗存在。
- 旧的右侧常驻 `detail-panel` tabs 被移除或不再作为三栏常驻结构。
- `app.js` 包含视图历史、快捷键、项目关闭、搜索、新项目文件读取、登出清理等函数。
- CSS 包含顶栏、两栏 Shell、折叠侧栏、边缘唤起、居中对话、搜索弹窗等布局 hook。

现有后端 WebUI 测试应继续通过。若静态测试原先强绑定三栏结构，需要同步改成新 Shell 断言。

## 验收标准

本阶段完成后，应满足：

- 登录后进入 Codex-like 主 Shell。
- 用户可以通过 `文件 -> 新项目` 或侧栏 `新项目` 导入 `spec.md`、`checklist.md`、可选 `index.html` 创建项目。
- 用户可以搜索并切换项目。
- 用户可以折叠侧栏，鼠标移动到左边界可临时唤起侧栏。
- 用户可以用顶部菜单、快捷键完成新窗口、新项目、关闭项目、设置、登出、退出。
- 用户可以用 `Ctrl+[` 和 `Ctrl+]` 在对话、设置、状态、预览、报告、审计、审批视图之间导航。
- 运行流程仍可完成 Mock LLM 写入 `index.html`，并能查看或下载产物。
- 测试通过：`python -m unittest discover -s tests -v`。
