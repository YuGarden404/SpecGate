# 静态 HTML 任务

请生成一个“上下文压缩观察页”，用于展示 SpecGate 在大上下文场景下如何保留关键需求并清理冗余信息。

页面必须包含：
- 上下文压缩
- 工具结果清理
- Gate 反馈
- Runtime Feedback

请使用内联 CSS，让页面结构清晰；交付物只需要更新 `index.html`。

## 无关背景材料

以下段落用于制造上下文预算压力，不包含额外有效需求。

段落 001：这是一段无关背景材料，用于观察上下文压缩策略是否保留关键任务要求。它描述一个虚构团队每周整理会议纪要、归档图片素材、统计表格字段，并把旧项目中的备注迁移到新的知识库。该段内容不改变页面需求。

段落 002：这是一段无关背景材料，用于观察工具结果清理是否避免保留过长的旧输出。材料提到若干课程、会议、排期、颜色偏好和文件命名习惯，但这些只是噪声，不应覆盖前文列出的页面关键词。

段落 003：这是一段无关背景材料，用于模拟真实任务里夹杂的历史讨论。讨论者提到不同浏览器窗口、临时截图、演示文稿和备忘录，但没有提出新的功能要求，也不影响交付文件范围。

段落 004：这是一段无关背景材料，用于给 Gate 反馈保留测试制造干扰。文本包含大量普通叙述，例如团队成员如何复查日志、如何整理待办、如何删除过期缓存，以及如何命名阶段性报告。

段落 005：这是一段无关背景材料，用于制造 Runtime Feedback 周围的上下文噪声。它反复说明这些信息只是背景说明，不是新的页面需求；最终页面仍应聚焦上下文压缩、工具结果清理、Gate 反馈、Runtime Feedback。
